Monarca Camellia fue una landing page sencilla para una marca que, por lo que pude ver, esta relacionada con tes. No fue un proyecto con una gran historia tecnica detras, pero si tuvo algo que me gusta recordar: fue una muestra mas de lo practico que puede ser Astro cuando el sitio correcto solo necesita HTML, CSS y una buena build.

El diseño tenia una presencia curiosa, con tonalidades azules fuertes y una identidad visual marcada. Era una pagina de marca, mas que una aplicacion: su trabajo era presentar una atmosfera, ordenar contenido y dejar una primera impresion clara.

## Desarrollo y reto tecnico

Construí la landing con Astro y Tailwind CSS. La parte mas particular no estuvo tanto en la implementacion del sitio, sino en el despliegue. El cliente tenia su propio servidor en Bluehost y lo unico que recibí fueron accesos FTP. A partir de eso generé la build y subi manualmente los archivos al servidor.

Ese flujo me recordo por que Astro me gusta tanto para este tipo de proyectos. Cuando la salida es estatica, el resultado puede vivir practicamente en cualquier servidor: no necesita procesos activos, llaves privadas, runtime especial ni configuraciones pesadas. Es una carpeta de archivos lista para publicarse.

El detalle visual que mas trabajo dio fue el uso de patrones del logo. No fue una dificultad enorme, pero si requeria cuidar que la marca se integrara bien en la composicion sin sentirse forzada.

## Tecnologias utilizadas

1. Astro.js
2. Tailwind CSS
3. FTP
4. Bluehost

## Resultado

Monarca Camellia quedo como una landing sencilla, visual y directa, publicada en el servidor propio del cliente. Para mi representa uno de esos proyectos donde la virtud esta en no complicar lo que no necesita complicarse: construir, generar, subir y dejar funcionando.
