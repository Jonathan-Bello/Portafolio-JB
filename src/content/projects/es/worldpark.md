Worldpark es un sitio Gatsby para presentar desarrollos industriales, naves, áreas comerciales y ubicaciones estratégicas. Cuando participé en el proyecto, la base ya estaba iniciada, pero había secciones con bastante carga de información que necesitaban organizarse y crecer de forma consistente.

## Desarrollo y reto técnico

Mi participación estuvo enfocada principalmente en la sección de desarrollos. Ahí se organizaban ubicaciones como Lerma, Tultitlán o Monterrey, junto con fichas técnicas, superficies, direcciones, PDFs, videos y contenido comercial para cada desarrollo.

El reto técnico fue manejar un sitio con volumen real de contenido sin volverlo inmanejable. Había que estructurar datos, reutilizar componentes, conectar recursos descargables y presentar información técnica de forma clara. También trabajé con videos alojados en Cloudinary; varios archivos llegaban pesados, así que aproveché transformaciones de Cloudinary para bajar calidad y peso sin perder una presentación visual aceptable.

Fue un proyecto laborioso porque combinaba contenido repetitivo, recursos multimedia y detalles comerciales que debían mantenerse consistentes entre páginas. Para mí, fue una buena experiencia de Gatsby aplicado a un sitio con catálogo real, no solo a una landing pequeña.

## Tecnologías utilizadas

1. Gatsby
2. React
3. Styled Components
4. Bootstrap
5. Google Map React
6. Swiper
7. Cloudinary
8. Gatsby Sitemap

## Resultado

El sitio funciona como plataforma de presentación comercial para desarrollos industriales, con catálogos, mapas, videos y contenido visual de apoyo. Para mi portafolio, Worldpark muestra experiencia en organización de información, manejo de recursos multimedia y mantenimiento de secciones con volumen real de datos.
