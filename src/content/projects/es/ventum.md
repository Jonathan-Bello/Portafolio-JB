Ventum es un sitio Gatsby heredado dentro de YGU. No lo desarrollé desde cero y mi participación fue mucho más limitada que en otros proyectos del portafolio. La entrada existe porque representa otro tipo de trabajo profesional: recuperar, mover y mantener proyectos antiguos de la empresa.

El sitio venía de una etapa previa donde varios repositorios estaban ligados al GitHub personal de un desarrollador anterior. Para no depender de accesos externos cada vez que hubiera que hacer un cambio, empecé a centralizar proyectos dentro de una organización propia de GitHub para la empresa.

## Desarrollo y reto técnico

Mi trabajo principal fue migrar o recuperar el proyecto hacia un entorno más controlado por YGU, hacer ajustes puntuales y mantenerlo disponible. Entre los cambios registrados estuvo actualizar números de contacto y WhatsApp, además de mover el repositorio para poder administrarlo sin depender del flujo anterior.

El reto no estuvo en crear funcionalidades nuevas, sino en entender un proyecto viejo, levantarlo, ubicar qué debía cambiarse y conservarlo funcionando. Este tipo de mantenimiento puede parecer menor, pero es importante cuando una empresa necesita mantener control sobre sus activos técnicos.

Ventum también sirve para marcar una diferencia clara dentro del portafolio: no todos los proyectos listados son sitios que construí desde cero. Algunos representan mantenimiento, migración, soporte o administración técnica de proyectos heredados.

## Tecnologías utilizadas

1. Gatsby
2. React
3. Styled Components
4. Ant Design
5. React Calendar
6. React Helmet
7. Google Analytics
8. Gatsby Sitemap

## Resultado

El sitio se mantuvo operativo después de moverlo a un flujo más controlado por la empresa y realizar cambios puntuales de contacto. Para mi portafolio, Ventum representa experiencia en migración y mantenimiento de proyectos heredados, más que desarrollo visual o funcional desde cero.
